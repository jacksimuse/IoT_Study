# Git 용어 3가지

1. Repository (저장소)
프로젝트가 있는 디렉토리나 저장 공간. "repo"로 줄여서 사용하기도 한다.
내 컴퓨터의 로컬 저장소가 될 수도 있고, 온라인 원격 저장소가 될 수도 있다.

2. Commit (커밋)
로컬 저장소의 코드/파일 수정 또는 생성/삭제 후 이를 원격 저장소에 push 하기 위한 전 단계 과정이다.
커밋하면, 그 시점의 저장소의 "스냅샷"을 찍어 프로젝트를 이전의 어떠한 상태로든 재평가 하거나 복원할 수 있는 체크포인트를 가질 수 있다.
하나 이상의 파일에 변경한 내용을 저장할 때마다 새로운 commit이 생성된다.

3. fork
남이 만든 오픈소스 등을 직접 변경하고 싶다면 먼저 내 계정에 그 프로젝트의 복사본을 만든 다음 이를 수정하게 되는데, 이 과정을 fork라고 한다.
fork한 저장소는 원본과 연결되어 있다. 즉, 원본에 변화가 생기면(새로운 커밋이 생기면) fetch나 rebase를 통해 fork 된 저장소에 반영할 수 있다.
